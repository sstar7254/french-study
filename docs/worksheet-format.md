# 프랑스어 연습지 포맷 규격서

이 문서는 PRD - 프랑스어 습득 세션을 구현하는 개별 연습지(HTML 세트)가 120세트 내내 지켜야 할 공통 규격을 규정합니다. PRD가 "무엇을 왜 배우는가"라면, 이 문서는 "연습지를 어떻게 만드는가"입니다. 1주차 세트1을 기준 구현체(reference implementation)로 삼습니다.

## 0. 상위 원칙 (제작 시 항상 우선)

- **문법 설명 금지.** 시드는 텍스트와 음성에 심어지는 노출 대상이지, 설명 블록으로 등장하지 않습니다. "~는 이런 뜻입니다" 식 해설을 넣지 않습니다.
- **1:1 대응 금지.** 프랑스어 단어에 한국어 뜻을 나란히 달지 않습니다. 뜻은 맥락과 유추로 도달하게 합니다. (퀴즈의 선택지처럼 유추를 유도하는 장치는 예외로 허용)
- **실물 소재만.** 교재형 창작 지문 대신 실제 영상/기사/노래/이미지를 씁니다. 단, dictée·짧은 Input 텍스트는 시드를 심기 위해 제작자가 직접 쓸 수 있습니다 (저작권 회피 목적 포함).
- **짧아도 뱉게 한다.** 모든 세트는 최소 1회의 산출(작문 또는 발화) 과제를 포함합니다.
- **자율 분량.** 코어와 확장을 분기로 분리합니다.

## 1. 파일 규격

| 항목 | 규격 |
|---|---|
| 형식 | HTML 단일 파일 (인라인 CSS·JS, 외부 의존성은 웹폰트만) |
| 파일명 | `{N}주차 세트{M} - 프랑스어 연습지.html` |
| 언어 속성 | `<html lang="ko">` (지시문은 한국어, 학습 대상은 프랑스어) |
| 반응형 | 모바일(~380px)까지 레이아웃 붕괴 없이 동작 |
| 저장 | localStorage/sessionStorage 사용 금지 (artifacts 제약). 상태는 세션 내 메모리에만 유지 |
| 접근성 | 키보드 포커스 가시화(`:focus-visible`), `prefers-reduced-motion` 존중, 버튼에 `aria-pressed` 등 상태 표기 |

## 2. 세트 골격 (블록 순서 고정)

모든 세트는 아래 순서를 따릅니다. Écho(노래) 블록은 선택적으로 Expansion 위치에 들어가며, dictée는 Input과 분기 사이에 독립 섹션으로 올 수 있습니다.

```
Header (테마 선언 + 모드 토글)
  └ Ouverture   : 유추 퀴즈 1문항 → 시드를 여는 정답
  └ Input       : 실물 소재 + 맥락 파악 질문 (요지 한 문장 필수)
  └ Dictée      : (선택) 제작 텍스트 3문장 받아쓰기 + diff 채점
  └ Pause       : "더 할까요? / 오늘은 그만" 분기
  └ Écho/Expansion : 시드 가지치기 또는 노래 3단 패턴
  └ Production  : 스토리텔링 프레임 산출 과제 (작문/발화)
  └ Clôture     : 메타인지 1 + 자유 피드백 1 + 답변 전체 복사
Footer (Bon courage + 세트 번호 N/120)
```

### 2.1 블록별 규격

**Header**
테마를 한국어 한 줄로 선언하고, 그 주의 시드가 무엇에 주목하는지 암시합니다("이번 주에는 ~에 주목해 봅니다"). 시드 이름 자체(présent, 빈도 표현)는 footer에만 노출하고 학습자에게 문법 용어로 들이밀지 않습니다.

**Ouverture**
2지선다 유추 퀴즈 1문항. 오답은 재치 있게, 정답은 그날의 시드를 여는 문장으로 설계합니다. 정답 공개(`.reveal`)가 다음 블록으로 넘어가는 서사 역할을 합니다. 정답이 "둘 다"인 함정형을 적극 활용합니다.

**Input**
실물 소재(영상 링크/기사 발췌/이미지) 또는 제작 텍스트 1~2개. 첫 질문은 **반드시 "한국어 한 문장 요지 요약"**입니다("전부 이해 못 해도 괜찮습니다" 문구 포함). 이어 사실/의견을 가르는 질문, 시드 표현을 눈으로 찾는 질문을 배치합니다. 사전 사용을 명시적으로 만류합니다.

**Dictée** (선택)
제작자가 쓴 프랑스어 문장 3개. 각 문장에 "듣기"·"아주 천천히" TTS 버튼과 입력칸, "채점" 버튼을 답니다. 채점은 단어 단위 diff로 정답/누락/오표기를 색으로 구분합니다(2.3 참조). 저작권 때문에 dictée 문장은 인용이 아니라 창작이어야 합니다.

**Pause (분기)**
"더 할까요? / 오늘은 그만" 두 버튼. 전자는 확장 블록을 펼치고, 후자는 확장을 건너뛰고 Production으로 스크롤합니다. 어느 쪽을 골랐는지 복사 출력에 기록합니다.

**Écho / Expansion**
두 가지 형태 중 하나.
- *노래 패턴* (기본 배합상 주 2회): 곡 링크 → "특정 표현이 몇 번 들리는지 세기" → "들리는 단어 전부 잡기(언니) / 무슨 노래일지 짐작(베베)" → "후렴 따라 부르고 녹음". 가사 원문은 싣지 않습니다.
- *가지치기 패턴*: Ouverture/Input의 시드 표현에서 어간·관용구·변형으로 확장하는 뱉기 과제.

**Production**
"자고 일어나니 ~가 되었다 치고" 계열의 스토리텔링 프레임 필수. 언니 모드는 4~5문장 + 시드 표현 2개 이상, 베베 모드는 2~3문장으로 지시문이 갈립니다. TTS로 자기 문장을 들어보는 보너스를 답니다.

**Clôture**
메타인지 정리 1문항("헷갈린 것 하나 / 자리 잡은 것 하나를 한국말로") + 자유 피드백 1문항 + "답변 전체 복사" 버튼. 복사 성공 시 확인 메시지를 노출합니다.

## 3. 인터랙션 컴포넌트 규격

재사용 가능한 JS 컴포넌트입니다. 신규 세트는 이 함수 시그니처를 그대로 씁니다.

| 컴포넌트 | 함수 | 규격 |
|---|---|---|
| 모드 토글 | `setMode('bebe'\|'eonni')` | `body[data-mode]` 전환. 지시문은 `.bebe-only`/`.eonni-only`로 분기. 기본값 언니 |
| 유추 퀴즈 | `quizN(btn, isRight)` | 선택 시 `.picked-right`/`.picked-wrong`, 정답 시 `.reveal.show`. 선택 로그는 `choiceLog`에 적재 |
| TTS | `speak(text, btn, rate)` | `lang='fr-FR'`, 프랑스어 보이스 자동 선택, 기본 rate 0.88, 느림 0.65~0.72. 재생 중 재호출 시 중단 |
| Dictée 채점 | `grade(i)` + `norm(w)` | `norm`으로 대소문자·악상·문장부호 무시 비교. 단어 단위 매칭 |
| 분기 | `openExpansion()` / `skipExpansion()` | 확장 표시/스킵 + 선택 기록 + 스크롤 |
| 녹음 | `toggleRec()` | MediaRecorder로 녹음→`<audio>` 재생. 실패 시 휴대폰 녹음 안내로 폴백 |
| 답변 수집 | `copyAll()` | `[data-answer]` 전부 + `choiceLog` + 모드/분기를 텍스트로 묶어 클립보드 복사. `execCommand` 폴백 포함 |

### 3.1 답변 태깅 규칙 (제출·피드백의 핵심)

학습자의 입력을 받는 모든 요소에는 `data-answer="라벨"`을 답니다. `copyAll()`이 이것을 순회해 제출 텍스트를 생성하므로, **태그가 없으면 피드백 루프에서 누락됩니다.** 라벨은 사람이 읽어 무슨 문항인지 알 수 있게 씁니다(예: `Q1 요지 한 문장`, `Dictée 2`, `Q7 나의 하루 (작문)`).

### 3.2 복사 출력 포맷 (recast 피드백 입력 규격)

`copyAll()`의 출력은 아래 구조를 유지합니다. Claude가 이 형식을 전제로 피드백하기 때문입니다.

```
=== {N}주차 연습지 ({M}) — {테마} ===
시드: {시드}
모드: 베베|언니
확장 분기: {선택}

[선택 로그들]
[data-answer 라벨]
답변 또는 (무응답)
...
→ 피드백 부탁해요!
```

## 4. 디자인 토큰 (고정)

Cahier(프랑스 공책) 아이덴티티를 120세트 내내 유지합니다. 세트별로 바꾸지 않습니다.

| 토큰 | 값 | 용도 |
|---|---|---|
| `--paper` | `#f8f7f2` | 배경(공책 종이) |
| `--ruling` | `#dbe7f0` | 가로 괘선 |
| `--marge` | `#e8737e` | 세로 여백선(마르주), 블록 라벨 |
| `--ink` | `#20409a` | 프랑스어·손글씨(빅 볼펜 파랑) |
| `--correct` | `#2e7d4f` | 정답 |
| `--wrong` | `#c0392b` | 오답·누락 |

- **폰트**: 프랑스어/제목 = `Caveat`(손글씨), 본문/지시문 = `Gowun Dodum`. 이 페어링을 유지합니다.
- **배경**: Seyès 괘선(가로 반복선 + 좌측 세로 마르주)을 CSS gradient로 구현. 모바일에서 마르주 위치를 조정합니다.
- **블록 라벨**: 각 `section`은 `data-label="Ouverture"` 식 속성으로 상단에 프랑스어 태그를 띄웁니다(마르주 색 알약).

## 5. 콘텐츠 제작 규칙

- **Input 텍스트 난이도**: B1+~B2-. 배경지식 없이 읽히는 소재만. 지정학도 이야기 단위로.
- **소재 풀**: 캐나다/프랑스 문화, 여성의 이야기, 최근 이슈, 국가별 지정학 이야기. 스케줄이 아닌 경계. Phase 3+는 주 1회 이상 찬반 이슈 포함.
- **노래/T'choupi 비중**: 주 5세트 중 약 2세트(소리·발음 중심). 페이즈 상승 시 기사류 비중 증가.
- **저작권**: 노래 가사·기사 본문을 학습지에 복제하지 않습니다. 소재는 링크로 연결하고, 학습지에 싣는 프랑스어 문장(dictée, Input 창작문)은 직접 씁니다. 유튜브는 링크 깨짐 방지를 위해 검색 링크를 우선합니다.
- **지시문 톤**: 격려하되 굴종하지 않기. "틀려도 뱉는 게 이깁니다", "잡힌 만큼만" 식의 담담한 응원. 빈 감탄사("좋아요!") 지양.
