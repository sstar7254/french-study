# 피드백 루프 규격서

이 문서는 학습자의 **답변**과 Claude의 **피드백**이 다음 연습지에 반영되도록 하는 지속(persistence) 구조를 규정합니다. PRD가 "무엇을 배우는가", 포맷 규격서가 "연습지를 어떻게 만드는가"라면, 이 문서는 **"한 회차의 결과가 어떻게 다음 회차로 넘어가는가"**입니다.

## 0. 왜 필요한가 (문제 정의)

- 세션은 휘발성이다. 이 대화가 끝나면 Claude는 지난 회차를 기억하지 못한다.
- 연습지(HTML)는 상태를 저장하지 못한다 (artifacts 제약, localStorage 금지). 유일한 출력 경로는 `copyAll()` → 대화 붙여넣기다.
- 따라서 답변과 피드백이 다음 세트로 넘어갈 통로는 **리포지토리(git)뿐**이다. 대화·Claude 메모리에만 의존하면 PRD §9의 "연속성" 리스크가 그대로 실현된다.

이 문서의 구조는 그 통로를 파일로 못 박는다.

## 1. 2층 구조

| 파일 | 역할 | 성격 | 크기 |
|---|---|---|---|
| `log/S{NNN}.md` | 회차별 제출 원문 + 피드백 원문 보관 | append-only 아카이브 (한번 쓰면 수정 안 함) | 회차당 1개, 최대 120개 |
| `carryover.md` | 다음 세트 제작 전 **필독 상태판** | 매 피드백 후 갱신되는 살아있는 문서 | 항상 1개, 압축 유지 |

**분리 원칙**: `log/`는 전체 기록의 진실 원본이고, `carryover.md`는 그로부터 추출한 작은 요약이다. 다음 세트를 만들 때 Claude는 `carryover.md` **한 장만** 읽으면 되고, 근거를 파고들 필요가 있을 때만 해당 `log/S{NNN}.md`를 연다. 이 분리가 없으면 회차가 쌓일수록 제작 비용이 폭증한다.

### 1.1 세트 ID

- 전역 일련번호 `S001` ~ `S120`. 연습지 footer의 `Set N/120`과 1:1로 대응한다.
- 로그 파일명은 이 ID를 그대로 쓴다: `log/S001.md`.
- 주차/일(day) 표기는 파일 안에서 메타데이터로 병기한다 (예: `1주차 · Day 1`).

## 2. `log/S{NNN}.md` 규격

한 회차가 끝나고 피드백을 준 직후, Claude가 이 파일을 새로 만든다. 골격은 `log/_TEMPLATE.md`를 따른다.

```
# S001 — {테마}

- 날짜: 2026-07-13
- 세트: 1/120 · 1주차 · Day 1 · Phase 1
- 시드: présent / 빈도 표현
- 모드: 언니 · 확장 분기: 함

## 제출 (copyAll 원문)

    (학습자가 붙여넣은 copyAll 출력을 그대로. 손대지 않는다.)

## 피드백 (recast 원문)

    (Claude가 이 회차에 준 피드백을 그대로.)

## 추출 → carryover 반영

- 재노출 큐 추가: {표현/구조} — {왜}
- 오류 패턴: {유형}
- 자리잡음(졸업): {표현/구조}
- 소재·난이도 신호: {관찰}
```

**규칙**
- "제출"·"피드백" 섹션은 원문 보존. 요약·편집하지 않는다 (나중에 근거로 되짚기 위함).
- "추출" 섹션이 `carryover.md`로 흘러갈 항목의 대장(臺帳)이다. 여기에 적은 것만 상태판에 반영된다.
- 무응답·미제출 회차도 로그를 남긴다 (진도 공백 자체가 신호다).

## 3. `carryover.md` 규격 (핵심)

다음 세트 제작 직전에 **반드시 먼저 읽는** 파일. 매 피드백 후 갱신한다. 항상 압축 상태를 유지한다(졸업한 항목은 큐에서 빼서 §3.4로 옮긴다).

### 3.1 진도

마지막으로 완료한 세트, 다음 세트 번호, 현재 Phase/주차, 모드 경향을 한눈에.

### 3.2 재노출 큐 (가벼운 간격반복)

"반영"을 기계화하는 장치. 학습자가 걸린 표현/구조를 여기 넣고, 지정한 세트에서 Écho/Expansion/Input에 다시 심는다. 잘 나오면 졸업(§3.4)시키고, 또 틀리면 뒤로 미룬다.

| 컬럼 | 의미 |
|---|---|
| 표현/구조 | 재노출 대상 (예: `avoir/être 조동사 선택`, `à mon avis`) |
| 최초 | 처음 걸린 세트 ID |
| 상태 | `신규` / `대기` / `관찰중` |
| 다음 등장 | 심을 예정 세트 ID (예: `S004`) |
| 메모 | 어떻게 틀렸는지 한 줄 |

간격 기본값(느슨하게): 신규 → +2~3세트 뒤 1차 재노출 → 맞으면 +5세트 뒤 확인 → 또 맞으면 졸업. 틀리면 다음 세트로 당겨 재노출. 엄밀한 Leitner가 아니라 "잊을 만하면 다시"의 감각. PRD의 "느슨하고 재밌게" 원칙을 지킨다.

### 3.3 반복 오류 패턴

한 번이 아니라 여러 회차에 걸쳐 반복되는 오류 유형을 누적한다 (예: `형용사 성수일치 누락`, `조동사 혼동`). 개별 실수는 재노출 큐로, **패턴**은 여기로. 패턴이 잡히면 그 주의 시드 선택에 영향을 준다.

### 3.4 자리잡은 것 (졸업)

재노출 큐에서 두 번 연속 통과해 빠져나온 항목. 이걸 기록해야 **그만 심을** 수 있다 (과잉 반복 방지).

### 3.5 소재·난이도 캘리브레이션

- 선호/기피 소재 신호 (자유 피드백 칸에서 읽어낸 것)
- 난이도 신호 (요지 요약 정확도, "오늘은 그만" 빈도)
- 모드 경향 (베베/언니 선택 추이)
- 분량 경향 (확장 분기 선택률)

이 절이 소재 정책(PRD §4)의 "제작 시점 선택"을 데이터로 뒷받침한다.

## 4. 운영 루프 (갱신판)

PRD §7을 이 구조로 구체화한다.

```
1. 제작   ─ carryover.md 를 먼저 읽는다.
            · 재노출 큐에서 "다음 등장 == 이번 세트 ID"인 항목을 Écho/Expansion/Input에 심는다.
            · 반복 오류 패턴이 있으면 그 주 시드 선택에 반영한다.
            · 소재·난이도 신호로 소재/모드 기본값을 조정한다.
2. 학습   ─ 학습자가 브라우저에서 세트 수행.
3. 제출   ─ copyAll() → 대화 붙여넣기.
4. 피드백 ─ Claude가 recast 방식으로 피드백.
5. 기록   ─ ★ 새로 추가되는 고리 ★
            a) log/S{NNN}.md 생성 (제출 원문 + 피드백 원문 + 추출).
            b) carryover.md 갱신 (재노출 큐 이동/추가, 패턴 누적, 졸업 처리, 진도 +1).
            c) 커밋 & 푸시.
```

**5단계(기록)가 이 문서의 존재 이유다.** 이 단계를 건너뛰면 다음 세션의 Claude는 지난 회차를 알 방법이 없다.

### 4.1 제작 시 체크 (§6 품질 체크리스트에 추가)

- [ ] 제작 전에 `carryover.md`를 읽었는가
- [ ] "다음 등장 == 이번 세트"인 재노출 큐 항목을 실제로 심었는가
- [ ] 피드백 후 `log/S{NNN}.md`를 남기고 `carryover.md`를 갱신했는가

## 5. 프라이버시 주의

이 구조는 학습자의 답변과 피드백을 git 히스토리에 남긴다. 개인적인 내용을 쓸 수 있는 자유 작문/피드백 칸이 있으므로, 리포지토리를 공개로 전환할 때는 `log/`의 내용을 검토한다. 민감한 회차는 로그에서 제외하거나 익명화할 수 있다 (그 경우 carryover의 추출 항목만 남긴다).
